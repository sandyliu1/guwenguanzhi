# GuwenLearner 架构说明

## 当前架构

### 技术栈

纯前端静态应用，无后端、无构建工具、无框架。

- HTML / CSS / 原生 ES Module JS
- 部署：GitHub Pages（自动，push 即生效）
- 本地开发：`npx serve . -p 3456`

### 文件结构

```
GuwenLearner/
├── index.html          # 页面骨架，包含导航、正文区、两个抽屉
├── style.css           # 全部样式
├── src/
│   ├── main.js         # 入口：初始化、导航构建、搜索、文章切换、点击事件
│   └── analyzer.js     # 核心：词汇匹配、高亮 HTML 生成
├── data/
│   ├── articles.json   # 文章列表（含富字段）
│   ├── shici.json      # 47 常考实词词库
│   └── xuci.json       # 18 常考虚词词库
├── basicWords/         # 原始 Word 文档（词库来源）
└── articals/           # 原始文章 Word 文档
```

### 数据层

所有数据在页面加载时通过 `fetch` 一次性拉取，存在内存里。

**articles.json 字段说明**

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✓ | 文章标题 |
| `author` | ✓ | 作者 |
| `period` | ✓ | 时期（导航第一层，如"春秋"） |
| `category` | ✓ | 类别（导航第二层，如"周文"） |
| `group` | ✓ | 分组（导航第三层，如"西周日暮"） |
| `content` | ✓ | 正文 |
| `keyPhrases` | 可选 | 特定句子里的重点词，标橙色实心 |
| `keySentences` | 可选 | 重点句，标橙色虚线下划线 |
| `annotations` | 可选 | 生僻字注释，紫色高亮 + 注音 + 悬停 tooltip |
| `background` | 可选 | 背景介绍，点「背景」按钮从右侧滑出 |

富字段均为可选，不写不影响功能。

### 高亮层级

优先级从高到低：

| class | 触发 | 视觉 |
|-------|------|------|
| `mark-key` | keyPhrases 指定词 | 橙色实心 |
| `mark-annotation` | annotations 指定字 | 紫色浅色 + 注音 |
| `mark-shici` | shici.json 匹配 | 黄色 |
| `mark-xuci` | xuci.json 匹配 | 绿色 |
| `key-sentence` | keySentences 指定句 | 橙色虚线下划线（叠加） |

匹配在 `analyzer.js` 的字符数组层面完成，先占位再生成 HTML，避免标签嵌套冲突。

### 交互

- 点击高亮词 → 底部抽屉展示用法释义 + 本文出现句
- 点击「背景」按钮 → 右侧抽屉展示背景信息
- 导航搜索框 → 实时过滤文章名，自动展开匹配的分组
- Esc / 点击外部 → 关闭抽屉

## 扩展时需要注意的问题

### 1. 数据体积

目前一次性加载所有文章。220篇预计 JSON 体积 300KB–1MB。

- **当前**：可接受，不需要处理
- **超过 1MB 时**：考虑拆分为 `articles-index.json`（只含标题/导航字段）+ 按需加载单篇内容

### 2. articles.json 手工维护

220篇手工编辑 JSON 容易出错（中文引号、漏逗号等）。

- **建议**：写一个 Python 脚本，从 Word/txt 批量生成 JSON，减少手工操作
- **注意**：`keySentences` 里不能直接写含中文引号 `" "` 的字符串，需拆开或转义

### 3. 导航性能

导航是一次性用 JS 构建所有 DOM 节点。220篇问题不大，但如果增加到 500+ 篇需要考虑虚拟列表。

### 4. 词库扩展

目前 shici 47词、xuci 18词，均为固定列表。

- 如果词库扩展到几百词，匹配逻辑（`analyzer.js` 的双重循环）可能变慢
- **优化方向**：改为 Aho-Corasick 多模式匹配，或限制每篇只匹配前 N 个词

### 5. 注音和注释的维护

`annotations` 目前是文章级别的，每篇单独维护。如果多篇文章出现同一个生僻字，需要重复写。

- **将来可以**：建一个独立的 `annotations.json` 全局字典，文章里只引用字，不重复写释义

### 6. 移动端适配

目前有基本的响应式布局，但抽屉宽度（340px）在小屏手机上偏窄，搜索框在导航隐藏时无法使用。需要专门测试和调整。
